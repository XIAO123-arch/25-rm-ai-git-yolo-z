# 个人反思

请至少回答以下问题。

## 1. 程序整体流程是什么？

程序通过命令行接收参数，区分文件夹、图片文件、视频文件三种输入类型；OpenCV读取图片或者视频帧，加载YOLO‑v8权重模型完成目标检测；再依照`--max‑det`参数，按照置信度由高到低筛选检测框；之后绘制目标方框与置信度文本，控制台输出检测统计信息；最后将处理完毕的文件保存至指定文件夹，视频文件逐帧处理后合成视频文件输出。

## 2. YOLO 在项目中负责什么？

YOLO‑v8模型接收图像数据，进行深度学习推理，识别画面中的无人机目标，输出检测框坐标、置信度数值以及物体类别编号；检测结果交给后续代码排序过滤，是整个项目实现目标识别的核心模块。

## 3. 多目标过滤是怎么实现的？

读取模型输出的全部检测框数据，提取每个目标的置信度，使用索引翻转方式完成置信度从高到低排序，兼容旧版PyTorch版本；读取控制台传入`--max‑det`参数，只保留置信度排名靠前的目标数量，舍弃其余检测框，之后仅绘制筛选完毕的目标框，同时统计并打印目标总数和类别数量，完成多目标过滤。

## 4. AI 帮你完成了哪些部分？

借助AI尽力detect.py整体代码，划分图片读取、YOLO推理、绘制方框、文件保存四个核心模块；生成`filter_boxes`、`print_statistics`、`run_video`、`run_folder`函数代码；改写main函数实现三分支逻辑；教会我Git‑Bash命令、规范Git提交注释格式；给出每条功能对应的测试指令，排查路径层级报错，指导我按照3‑4天任务要求迭代开发。并且整理全部对话记录。

## 5. 哪些代码是你自己理解后修改的？

读懂`run_image`函数执行流程，核对图片读取、模型推理、绘制方框、文件保存四个环节；结合ai修改run‑image保存逻辑，让文件自动继承原图名称避免批量覆盖；调试命令行参数传递，将`--conf`、`--max‑det`参数传入视频和批量图片函数；编写Git提交注释，核对每条指令运行效果，测试参数修改之后程序变化。

## 6. 遇到的最大问题是什么？怎么解决的？

开发期间最主要问题是看不懂代码，后在AI帮助下尽力理解了代码，大致明白了程序逻辑。
文件夹路径层级错误，我经常停留在外层文件夹运行代码，程序找不到detect.py、素材图片和模型文件，代码直接抛出文件不存在报错。
之后我学会先用`ls`确认当前目录，执行`cd rm‑yolo‑demo‑starter`进入项目根目录再运行指令；并且借助Path工具统一相对路径，代码自动拼接项目根路径，彻底解决路径层级问题。
