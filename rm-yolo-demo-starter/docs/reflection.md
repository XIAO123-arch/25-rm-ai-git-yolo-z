# 个人反思

请至少回答以下问题。

## 1. 程序整体流程是什么？

程序先读取本地图片文件，加载YOLO‑v8权重模型；模型完成目标检测推理，获取目标坐标、置信度；之后绘制检测框与类别标签；最后将画框完成的图片保存至指定路径。

## 2. YOLO 在项目中负责什么？

YOLO‑v8模型接收读取后的图像数据，输出检测框坐标、置信分数、物体类别；把检测结果传给绘图函数，为后续画框过滤提供数据，是整个项目核心识别模块。

## 3. 多目标过滤是怎么实现的？

读取全部检测结果，按照置信度数值从高到低排序；读取命令行参数`--max‑det`，只保留置信度靠前的指定数量目标，其余目标舍弃，再绘制图像。

## 4. AI 帮你完成了哪些部分？

借助AI划分代码四大功能模块，添加代码注释；学习Git提交命令，解决GitHub网络推送失败问题；梳理项目每日任务流程，完善文档文本。

## 5. 哪些代码是你自己理解后修改的？

读懂run_image函数整体流程，在图片读取、模型推理、绘制方框、文件保存四处手动添加注释，划分代码模块。

## 6. 遇到的最大问题是什么？怎么解决的？

运行代码后找不到输出的outputs文件夹，一开始不清楚项目相对路径规则，图片保存失败；查阅代码中`resolve_output_path`路径解析函数，确认文件夹生成规则，代码会自动创建outputs目录，重新运行程序，成功生成输出图片。

## 7. 如果继续改进，你会做什么？

接下来添加视频检测功能，实现批量图片文件夹处理；完善`--conf`和`--max‑det`参数测试；补充统计文本，输出检测物体数量信息，完成第三天、第四天任务。
